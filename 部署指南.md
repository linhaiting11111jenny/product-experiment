# GitHub Pages 部署指南

本指南将帮助你将实验页面部署到GitHub Pages，使其可以通过公开URL访问，以便在Credamo等平台中使用。

## 📋 部署步骤

### 步骤 1：在 GitHub 上创建新仓库

1. 访问 https://github.com/new
2. 填写仓库信息：
   - **Repository name**: `product-experiment`（或你喜欢的名字）
   - **Description**: `商品浏览实验页面`
   - **Visibility**: 选择 Public（GitHub Pages 免费版需要公开仓库）
   - ⚠️ **不要勾选** "Add a README file"、"Add .gitignore"、"Choose a license"
3. 点击 "Create repository"

### 步骤 2：连接本地仓库到 GitHub

在终端中执行以下命令（将 `你的用户名` 替换为你的 GitHub 用户名）：

```bash
git remote add origin https://github.com/你的用户名/product-experiment.git
git branch -M main
git push -u origin main
```

**注意**：如果 GitHub 要求身份验证，你可能需要：
- 使用 Personal Access Token（而不是密码）
- 或者配置 SSH 密钥

### 步骤 3：启用 GitHub Pages

1. 在 GitHub 仓库页面，点击 **Settings**（设置）
2. 在左侧菜单中找到 **Pages**
3. 在 "Source" 部分：
   - 选择 **Deploy from a branch**
   - Branch 选择 **main**
   - Folder 选择 **/ (root)**
4. 点击 **Save**（保存）

### 步骤 4：等待部署完成

- GitHub Pages 通常需要 1-2 分钟来部署
- 刷新 Settings → Pages 页面，你会看到绿色的部署成功提示
- 你的页面将在以下 URL 可用：
  ```
  https://你的用户名.github.io/product-experiment/index.html
  ```

## 🔗 部署后的URL

部署完成后，你的页面将在以下URL可用：

```
https://你的用户名.github.io/product-experiment/index.html
```

**注意**：将 `你的用户名` 替换为你的GitHub用户名，`product-experiment` 替换为你的仓库名。

关于如何在Credamo中配置和使用这些链接，请查看 [Credamo配置指南.md](Credamo配置指南.md)。

## 🧪 测试你的部署

1. 访问你的 GitHub Pages URL
2. 测试不同条件：
   - 添加 `?condition=rating` 查看星级版本
   - 添加 `?condition=aigrs_neutral` 查看中立AIGRS版本
   - 添加 `?condition=aigrs_positive` 查看正向AIGRS版本
3. 测试功能：
   - 点击"查看原始评论"
   - 点击"加载更多评论"
   - 点击"完成浏览，返回问卷"
   - 打开浏览器控制台，检查 localStorage 中的 `experimentData`

## 📝 后续更新

如果需要更新代码：

```bash
git add .
git commit -m "更新说明"
git push
```

GitHub Pages 会自动重新部署（通常需要 1-2 分钟）。

## ❓ 常见问题

### Q: 页面显示 404？
A: 检查：
- 是否已启用 GitHub Pages（Settings → Pages）
- 是否选择了正确的分支（main）和文件夹（/root）
- 等待 1-2 分钟让部署完成
- 确保 URL 路径正确（包括 `/index.html`）

### Q: 如何查看部署状态？
A: 在仓库页面点击 **Actions** 标签，可以看到部署历史。

### Q: 可以自定义域名吗？
A: 可以！在 Settings → Pages → Custom domain 中设置。

### Q: 数据会保存在哪里？
A: 实验数据会保存在用户的浏览器 localStorage 中，并通过 postMessage 或 URL 参数传回 Credamo。
